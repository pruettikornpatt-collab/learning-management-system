9. ข้อสังเกตเชิงบริหาร

Accountability by Design — ทุกระบบของ J&T ถูกออกแบบมาให้ระบุความรับผิดชอบได้ทุกจุด (PDA scan ทุก handoff, ใบสั่งงานเป็นหลักฐาน, รหัสปรับ DP) แทนที่จะพึ่ง goodwill

Cost of Poor Quality ที่มองเห็นได้ — ทั้ง workflow พัสดุไม่มี AWB เป็นต้นทุนที่เกิดจาก quality ที่ปลายทางต้นทาง ถ้า upstream labeling ถูกต้องตั้งแต่แรก ทีม Store ทั้งหมดไม่จำเป็นต้องมี

Escalation Structure ที่ชัดเจน — ปัญหาเล็กแก้ที่หน้างาน / กลางให้สาขาต้นทาง confirm / ใหญ่รอ HQ อนุมัติ / VIP ให้สาขาปลายทางตัดสิน แต่ละระดับมีหน้าที่ไม่ซ้อนกัน ป้องกันการใช้ดุลยพินิจผิดระดับ

1. Distribution Network Structure

J&T operates a three-tier network. Currently 11 Gateways and 6 HUBs nationwide.

TierRoleCan dispatch to
GW (Gateway)Largest sorting center, highest capacityGW, HUB, Branches
HUBMid-tier, offloads GW volumeGW, HUB, Branches (upcountry only)
BranchCustomer-facing pick-up / delivery pointEnd customers

Key rules: GW→GW trucks cannot be turned back under any circumstance · If a branch truck contains >30% problematic parcels, the entire truck may be returned — the branch is responsible for repacking.

2. Core Operations

Inbound

Every parcel passes through a DWS machine (auto dimensioning, weighing, scanning). SPK runs 22 machines simultaneously, sorting parcels by destination zone automatically. Unreadable labels circulate on a Reverse Conveyor until staff redirect them manually.

AutoPacking / Sack House

Parcels ≤3 kg from branches are sorted automatically by destination. Ring D (the largest) has 36 machines. KPI: 1,600 pcs/hr. Spherical or thin items go to the Fliptray line instead (Fliptray 2 handles 4 zones).

Outbound

Staff manually sort parcels by destination line and scan each one via PDA before loading. Manual operation was deliberately chosen over automation because peak volume exceeds machine capacity.

3. Tracking & Accountability (JMS + PDA)

Every handoff is PDA-scanned, creating a timestamped record in JMS. If a parcel is lost or damaged, the responsible party is identifiable immediately.

Origin branch → Origin GW → Destination GW → Destination branch → Signature (Every node has inbound + outbound scans as accountability proof)

4. Quality Control & KPIs

QC comprises 5 units (Call Center · CCTV · Data · Packing · Store), covering three dimensions: real-time, retrospective, and customer-facing.

On-time inbound/outbound≥ 99.2%

9. Management Observations

Accountability by Design — Every system J&T has built (PDA scans at every handoff, work orders as evidence, DP penalty codes) creates an auditable trail pinpointing where problems occur, replacing debate with data.

Visible Cost of Poor Quality — The entire no-AWB workflow represents the cost of inadequate upstream labeling. If origin branches attached labels correctly from the start, the Store team's entire workload would not exist.

Clear Escalation Structure — Small issues resolved on the floor / medium issues require origin-branch confirmation / large issues await HQ approval / VIP decisions belong to the destination branch. No level overlaps, preventing judgment calls at the wrong tier.

1. 配送网络结构

极兔速递采用三级配送体系，目前全国共有 11 个分拨中心（GW）和 6 个中转中心（HUB）。

层级职能可发往
GW（分拨中心）最大分拣中心，设施最完备GW、HUB、网点
HUB（中转中心）次级节点，分担 GW 压力GW、HUB，省外可发至网点
网点（Branch）贴近客户的收派件窗口终端客户

关键规定：GW→GW 运输车辆不得拒收退回 · 若网点发来的车辆中问题件超过 30%，可整车退回，由网点自行重新打包。

2. 核心作业流程

进港（Inbound）

所有包裹须经过 DWS 设备（自动体积测量、称重、扫码），SPK 共配置 22 台同步运行，按目的地自动分拣。面单无法识别的包裹在反向传送带上循环，由人工辨认后归位。

自动装包 / 装包区

来自网点的袋装包裹（≤3 kg）拆袋后按目的地自动分拣。D 环线规模最大，配备 36 台设备。KPI：1,600 件/小时。圆形或薄型包裹另走 Fliptray 线（Fliptray 2 对应 4 个分区）。

出港（Outbound）

人工按目的地线路分拣，每件包裹装车前须经 PDA 扫描。此环节有意采用人工——经测试，自动化设备无法应对高峰时段的瞬时件量，曾导致包裹掉落及 SLA 不达标。

3. 追踪与责任体系（JMS + PDA）

每次交接均通过 PDA 扫描，在 JMS 系统中生成带节点标识的时间戳。一旦出现丢失或破损，可立即追溯到责任方。

始发网点 → 始发分拨中心 → 目的分拨中心 → 目的网点 → 签收 （每个节点均有进出港扫描记录，作为责任凭证）

4. 质控部门（QC）与 KPI

QC 下设 5 个单元（呼叫中心 · CCTV · 数据 · 包装 · 仓储），覆盖实时监控、历史追溯、客户服务三个维度。

进出港及时率≥ 99.2%
错分拣< 800 件/天
落扫率< 0.4%
清场达标率< 1%

5. 客服热线 1470

AI 系统根据包裹最新轨迹状态，将来电直接转接至对应节点。所有来电先经过总部，若目的节点坐席占线，由总部受理并通过后台转交跟进。分拨中心可直接联系网点，无需经过大区。

SLA：曼谷件须在 48 小时内处理完毕 · 滞留超 36 小时后可提交加急派送申请。

9. 管理层观察

责任体系内嵌于设计之中 — 极兔的每套机制（PDA 全程扫描、工单即证据、DP 罚款代码）均在构建可追溯的责任链，以数据替代争议。

质量成本清晰可见 — 无面单件的整套处理流程，本质上是上游贴单质量不达标的代价。若始发网点从源头保证面单准确，仓储（Store）团队的工作量将不复存在。

升级处理机制层次分明 — 小问题现场解决 / 中等问题由始发网点确认 / 大问题等待总部批准 / VIP 件由目的网点决策。各层级职责互不重叠，防止在错误层级行使判断权。
