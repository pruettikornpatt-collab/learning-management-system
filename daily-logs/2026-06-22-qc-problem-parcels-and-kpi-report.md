### 1. **โครงสร้าง Presentation แผนก QC**

แผนก QC ของ SPK รับผิดชอบงานหลัก 2 กลุ่ม ซึ่งคนมักสับสนว่าเป็นเรื่องเดียวกัน แต่จริงๆ แยกกันชัดเจน:

| หัวข้อ | จีน | ความหมาย | ฝ่ายที่ดูแล |
|---|---|---|---|
| 01 พัสดุมีปัญหา | 问题件 | พัสดุที่ **ส่งต่อไม่ได้ทันที** (รู้ว่าเป็นของใครแต่มีปัญหา) | Packing |
| 02 พัสดุไม่มี AWB | 无头件 | พัสดุที่ **ระบุเลขใบนำส่งไม่ได้** (ไม่รู้ว่าเป็นของใคร) | Store |

> **จุดต่างสำคัญ:** พัสดุมีปัญหา = มีเลข AWB แต่ส่งต่อไม่ได้ / พัสดุไม่มี AWB = ไม่มีเลขให้ส่งต่อตั้งแต่แรก

### 2. **⭐️ พัสดุมีปัญหา (问题件) — หัวข้อใหม่**

**นิยาม:** พัสดุที่ไม่สามารถส่งต่อได้ทันที

**ความสำคัญของการจัดการพัสดุมีปัญหา:**
- a. ลดความเสียหายที่อาจเกิดขึ้น
- b. ลดจำนวนพัสดุตกค้าง
- c. ลดจำนวนการเคลมพัสดุสูญหาย
- d. เพิ่มประสิทธิภาพในการทำงาน

**สาเหตุพัสดุมีปัญหา 4 ข้อ:**
1. บรรจุภัณฑ์ไม่ได้มาตรฐาน
2. การจัดเรียงพัสดุไม่ได้มาตรฐาน
3. การทำงานที่ไม่ได้มาตรฐาน
4. ใบนำส่งมีปัญหา / ไม่ชัดเจน

**ประเภทพัสดุมีปัญหา 2 กลุ่ม:**

| 1. พัสดุเสียหาย | 2. พัสดุไม่เสียหาย |
|---|---|
| พัสดุแตกเสียหาย | 2 ใบนำส่ง |
| ผลไม้เน่าเสีย | ใบนำส่งชำรุด / ไม่ชัดเจน |
| | พัสดุเปรอะเปื้อน |
| | สินค้าต้องห้ามฝากส่ง |

**ขั้นตอนการจัดการพัสดุมีปัญหา:**
```
1. แยกประเภทพัสดุมีปัญหา
        ↓
2. ลงทะเบียนพัสดุมีปัญหาทุกชิ้น 100%
        ↓
3. รีแพ็คพัสดุ
        ↓
4. แยกประเภทให้ชัดเจน:
   - พัสดุรอตอบกลับ (พัสดุเสียหาย)
   - พัสดุส่งออกตามรอบ
```

### 3. **พัสดุไม่มี AWB (无头件) — ทบทวน**

(รายละเอียดเต็มอยู่ใน log 2026-06-18) สรุปจาก Presentation วันนี้:

**สาเหตุ 4 ข้อ:** ไม่มีใบนำส่งจากสาขา / ใบหลุดระหว่างขนส่ง / ใบไม่ชัดเจน / พัสดุหลุดออกจากบรรจุภัณฑ์

**Flow:** แยกประเภท → จับคู่ → ลงทะเบียน (ข้อมูล 11 รายการ + ถ่ายรูป ≥3 รูปจากบรรจุเดิม) → จัดเก็บเข้าคลังตามวันเวลา/ประเภท/มูลค่า

**ขั้นตอนการส่งต่อ (2 เส้นทาง):**
```
เส้นทาง A (สาขายืนยันก่อน):
สาขายืนยันมาในระบบ → SPK ตรวจสอบ → ลงทะเบียนพัสดุมีปัญหา → ส่งต่อ/ตีกลับ

เส้นทาง B (SPK จับคู่เอง):
ตรวจสอบพัสดุตกค้าง → จับคู่พัสดุ → ลงทะเบียน → ส่งต่อ/ตีกลับ
```

**การแพ็คส่งออก:**
- บรรจุภัณฑ์ภายนอกชำรุด → เปลี่ยนกล่อง/ซองใหม่ขนาดเหมาะ / ของหลายชิ้นห่อบับเบิ้ลแยกแต่ละชิ้น
- บรรจุภัณฑ์ภายนอกไม่ชำรุด → เติมวัสดุกันกระแทก (บับเบิ้ล/โฟม/กระดาษลูกฟูก)

### 4. **KPI ทั้ง 5 ของ QC + ตัวเลขจากรายงานจริง**

| # | KPI (จีน) | นิยาม | เป้าหมาย | ผลจริงล่าสุด |
|---|---|---|---|---|
| 1 | 进出港及时率 (ขาเข้า/ขาออกทันเวลา) | สแกนถึง+นำส่งออกภายในรอบรถ (ถึง +60 นาที) | ≥ 99.2% | เข้า **98.11%** / ออก **98.91%** (06-02) |
| 2 | 错分拣 (คัดแยกผิด) | พัสดุถูกคัดแยกไปผิดปลายทาง | < 800 ชิ้น/วัน (0.28%) | รวม 17,899 ชิ้น (2 มิ.ย.) ⚠️ เกิน |
| 3 | 落扫 (หลุดสแกน) | พัสดุไม่ถูกสแกนครบทุกสถานะ | < 0.4% | รวม **0.57%** (29 พ.ค.–4 มิ.ย.) ⚠️ เกิน |
| 4 | 清场 (เคลียคลัง) | เข้า 13:00–04:00 ต้องออกก่อน 06:00/08:00/11:00 | < 1% | รวม **1.92%** (06-05) ⚠️ เกิน |
| 5 | 中转超时 (Transit ล่าช้า) | เซ็นรับล่าช้าเกิน SLA | (รอยืนยัน) | SPK_GW เข้า 7.37% / ออก 14.49% |

**รายละเอียด KPI 1 (进出港及时率):**
- ขาเข้า = พัสดุภายในเขต งาน A1
- ขาออก = พัสดุส่งออกนอกเขต ไป GW อื่น
- Logic = อ้างอิงเวลาถึง +60 นาที จนถึงรอบรถที่ใกล้ที่สุด
- จากรายงาน 06-02: SPK_GW ขาเข้า 98.03% / ขาออก 98.86% — **ยังต่ำกว่าเป้า 99.2%** ทั้งคู่

**KPI 2 (错分拣) สาเหตุ:** a. คัดแยกนำส่งผิด b. ติดลาเบลกระสอบผิด c. ทำงานไม่ได้มาตรฐาน

**KPI 3 (落扫) Logic:** พัสดุเซ็นรับแล้วที่สแกนถึงที่ SPK แต่ไม่มีสถานะนำส่ง → SPK_GW อยู่ที่ 0.69% (สูงกว่าเป้า 0.4%)

**KPI 4 (清场):** ดึงข้อมูลที่ 6:40 / 8:40 / 11:40 (ช้ากว่า deadline 40 นาที) — วันที่ 06-05 พุ่งไป 1.92% เพราะ SPK_GW ค้าง 69,649 ชิ้น (3.02%)

## Key Insights

✅ **"พัสดุมีปัญหา" ≠ "พัสดุไม่มี AWB"** — พัสดุมีปัญหา (Packing ดูแล) คือรู้ว่าเป็นของใครแต่ส่งต่อไม่ได้ ส่วนพัสดุไม่มี AWB (Store ดูแล) คือไม่มีเลขให้ส่งต่อ ทั้งสองอย่างเป็น "งานซ่อมปัญหา" ของ QC แต่คนละต้นตอ

✅ **SPK_GW ยังไม่ถึงเป้าใน KPI หลายตัวพร้อมกัน** — ขาเข้า/ขาออกต่ำกว่า 99.2%, หลุดสแกน 0.69% (เป้า 0.4%), เคลียคลังพุ่ง 1.92% (เป้า 1%) ในวันที่ของจริง แสดงว่า SPK ในฐานะ GW ที่ใหญ่ที่สุด (HQ) กำลังแบกปริมาณงานสูงและเป็นคอขวด

✅ **เคลียคลัง 06-05 เด้งขึ้นผิดปกติ** — จาก ~0.95% เป็น 1.92% ในวันเดียว เพราะ SPK_GW ค้าง 69,649 ชิ้น สะท้อนว่า KPI ตัวนี้ swing แรงตามปริมาณพัสดุเข้าแต่ละวัน

✅ **ลงทะเบียน 100% เป็นกฎเหล็กของทั้งสองงาน** — ทั้งพัสดุมีปัญหาและพัสดุไม่มี AWB เน้น "ลงทะเบียนทุกชิ้น" ก่อนทำอย่างอื่น เพราะถ้าไม่ลงระบบ = พัสดุหายจาก tracking และโดนค่าปรับ

## Connection to Management

KPI 5 ตัวของ QC เป็นตัวอย่างที่ดีของ **management by exception** — แทนที่จะดูทุกพัสดุ ระบบตั้งเป้า (99.2% / 0.28% / 0.4% / 1%) แล้วเด้งเฉพาะจุดที่หลุดเป้าออกมาเป็นสีแดง ทำให้ผู้จัดการโฟกัสทรัพยากรไปที่ปัญหาจริงได้ทันที

ที่น่าสนใจคือ SPK_GW (ศูนย์ HQ ที่ใหญ่ที่สุด) กลับหลุดเป้าหลายตัวพร้อมกัน — สะท้อนหลัก **bottleneck / theory of constraints** ว่า throughput ของทั้งเครือข่ายถูกจำกัดด้วยจุดที่มีปริมาณงานหนาแน่นที่สุด การปรับปรุง SPK จึงให้ผลต่อภาพรวมมากกว่าการไปปรับศูนย์เล็กที่ทำได้เกินเป้าอยู่แล้ว

นอกจากนี้การแยก "พัสดุมีปัญหา" กับ "พัสดุไม่มี AWB" ออกเป็นคนละฝ่าย (Packing vs Store) คือการ **แบ่งงานตามต้นตอของปัญหา** ไม่ใช่ตามอาการ — ทำให้แต่ละทีมเชี่ยวชาญเฉพาะทางและแก้ปัญหาได้ลึกกว่า
