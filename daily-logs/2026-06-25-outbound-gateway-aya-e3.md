## Today's Focus
ลงทำ **Outbound Gateway** ที่คลัง E3 บนเส้น **AYA (Ayutthaya GW)** ซึ่งเป็นเส้นที่พัสดุเยอะที่สุด — เรียนรู้ 3 หน้าที่หลัก: ปัดพัสดุจาก line หลักเข้าสายพานรอง, ยิง PDA ก่อนขึ้นรถ, และจัดของบนรถ

## What I Learned

### 1. **เส้น AYA = เส้นที่พัสดุเยอะที่สุด (มี 4 ช่อง)**

> AYA = **Ayutthaya Gateway** รหัส GW **999014** (อ้างอิงรหัสปลายทางจาก log 6-22) — เป็นเส้นปลายทางที่ volume สูงที่สุดในขาออก E3 จึงมีงานเข้ามาต่อเนื่องตลอดเวลา

เพราะพัสดุเยอะมาก เส้น AYA จึงมีถึง **4 ช่อง** (รถ 4 คันรับพร้อมกัน) — ไม่ใช่ช่องเดียวเหมือนเส้นเล็ก

---

### 2. **3 หน้าที่หลักของ Outbound Gateway (เส้น AYA)**

| ลำดับ | หน้าที่ | งานที่ทำ |
|---|---|---|
| 1 | ปัดพัสดุเข้าสายรอง | คอยปัดพัสดุจาก **line หลัก** ให้เข้า **สายพานรอง** ที่ส่งต่อขึ้นรถ |
| 2 | ยิง PDA ก่อนขึ้นรถ | สแกนนำส่งพัสดุด้วย **PDA** ก่อนโหลดขึ้นรถ |
| 3 | จัดของบนรถ | เรียง/จัดพัสดุบนรถให้เป็นระเบียบ ใช้พื้นที่คุ้ม + ปลอดภัยระหว่างขนส่ง |

**Flow:**
```
line หลัก (พัสดุไหลรวมมา)
      ↓  (1) ปัดพัสดุเส้น AYA ออก
สายพานรอง → ส่งขึ้นรถ
      ↓  (2) ยิง PDA นำส่งทีละชิ้น
ขึ้นรถ
      ↓  (3) จัดเรียงของบนรถ
รถออกตามรอบ
```

- การยิง PDA ก่อนขึ้นรถ = สแกนนำส่งออกทีละชิ้น สอดคล้องกับ flow outbound ใน log 6-16 (สายพาน → คัดแยกตาม line → สแกน PDA → ขึ้นรถ → ปิดตู้)

---

### 3. **⭐️ เกณฑ์การปัด — Label + ความพร้อมของช่อง**

**ดูจาก Label:** ถ้า Label เป็น **H1** (= อยุธยา / AYA) → ปัดเข้าเส้น AYA

**แต่ไม่ใช่เห็น H1 แล้วปัดได้เลย** — คนปัดต้องเช็คว่า **ช่องของตัวเองพร้อมครบองค์ประกอบ** ก่อน:

| องค์ประกอบที่ต้องครบก่อนเริ่มปัด |
|---|
| 1. **รถมาจอดที่ช่องแล้วหรือยัง** |
| 2. **คนยิง PDA อยู่ประจำช่องไหม** |
| 3. **คนโหลดของขึ้นรถอยู่ไหม** |

> ครบทั้ง 3 อย่าง (รถ + PDA + คนโหลด) ถึงจะเริ่มปัดพัสดุลงช่องได้ — ถ้าปัดทั้งที่ยังไม่พร้อม ของจะกองค้างที่ช่องโดยไม่มีใครยิง/โหลด กลายเป็นคอขวด

---

### 4. **ทำไมเส้นเดียวต้องมีทั้ง "ปัด → ยิง → จัด"**

- **ปัด (1):** กันไม่ให้พัสดุเส้น AYA ไหลเลยจุดไปปนเส้นอื่น/ตกท้ายราง — เป็นด่านคัดเข้าเส้นตัวเอง
- **ยิง PDA (2):** สร้าง record นำส่ง + ยืนยันว่าของชิ้นนี้ขึ้นรถคันถูก ก่อนปิดตู้
- **จัดของบนรถ (3):** เส้น volume สูงอย่าง AYA ถ้าจัดของไม่ดี รถเต็มเร็ว/ของเสียหาย/โหลดได้น้อยลง

## Key Insights

✅ **เส้น volume สูง = งานไม่ขาดมือ** — AYA มีของไหลตลอด ทำให้ทั้ง 3 หน้าที่ต้องทำพร้อมกันต่อเนื่อง จังหวะงานเร็วกว่าเส้นอื่น เป็นจุดที่กดดัน throughput ของขาออก E3 มากที่สุด (จึงต้องมีถึง 4 ช่อง)

✅ **การปัดไม่ใช่แค่ "อ่าน Label" แต่ต้อง "ดูความพร้อมปลายน้ำ"** — H1 บอกว่าของเป็นของ AYA แต่จะปัดได้ต่อเมื่อช่องมีรถ + คนยิง PDA + คนโหลดครบ เป็นการ **sync จังหวะกับ downstream** ไม่ให้ของกองค้าง = หลัก pull system / กันคอขวดที่ปลายช่อง

✅ **PDA ก่อนขึ้นรถ = จุดยืนยันสุดท้ายของขาออก** — เป็นด่านสแกนก่อนของจะออกจาก Gateway ถ้าพลาดตรงนี้ ของขึ้นรถผิดคัน/ไม่มี record นำส่ง จะตามยากตอนปลายทาง

✅ **จัดของบนรถมีผลต่อ capacity จริง** — ไม่ใช่แค่ยกขึ้น แต่การเรียงดีทำให้บรรทุกได้มากขึ้นต่อรอบ ลดจำนวนรอบรถ = ลดต้นทุนขนส่ง

## Connection to Management

เส้น AYA เป็นตัวอย่างของ **high-throughput bottleneck** ในขาออก — จุดที่ volume สูงที่สุดคือจุดที่กำหนดความเร็วของทั้งระบบ การจัดคนที่แข็งแรงและจังหวะงานเร็วไว้ที่เส้นนี้จึงให้ผลต่อภาพรวมมากที่สุด (สอดคล้องแนวคิด theory of constraints จาก log 6-22)

อีกมุมคือ **3 หน้าที่ = แบ่งงานตามขั้นของ value chain ย่อยๆ** (คัดเข้า → ยืนยัน → โหลด) ทำให้แต่ละจุดมีความรับผิดชอบชัด และของไม่หลุด flow ตั้งแต่ปัดจน PDA จนขึ้นรถ — การที่งานหนักแต่ "สนุก" เพราะงานต่อเนื่อง ยังสะท้อนหลัก **flow state / engagement** ที่งานที่มีจังหวะสม่ำเสมอและเห็นผลทันทีสร้างแรงจูงใจได้ดีกว่างานที่ขาดช่วง
