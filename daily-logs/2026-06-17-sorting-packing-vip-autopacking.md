### 1. **รหัสคัดแยก (Sorting Code)**

รหัสแบ่งเป็น 4 ส่วน:

| ส่วน | ตัวอย่าง | ความหมาย |
|---|---|---|
| 1 | A1 | คลัง/เขต |
| 2 | R15-12 | เส้นทาง/รอบรถ |
| 3 | 001A | สาขาปลายทาง |
| 4 | 122/3 | ลำดับย่อย |

- พัสดุต้องมีเลขคำสั่งงาน + ใบนำส่ง (18 หลัก)
- หลุดสแกนรับ = สแกนส่งออกไม่ได้
- รหัส `00` = อ่านไม่ได้ → ไปท้ายราง
- รหัส `*` = คัดแยกผิดตั้งแต่ต้นทาง หรือสาขาปิดงานไปแล้ว
- `Zx` = เส้นรอง

### 2. **Packing — กระบวนการจัดการพัสดุมีปัญหา (JMS)**

#### สถานะ "ระงับ" และ flow การตัดสินใจ
```
พัสดุมีปัญหา → สถานะ "ระงับ"
  ├── เสียหายทั้งหมด → รอ HQ approved → ปิดงาน
  ├── เสียหายบางส่วน → รอลูกค้า confirm
  │     ├── ลูกค้าเอา → ซ่อมแล้วส่งต่อ
  │     └── ลูกค้าไม่เอา → ตีกลับสาขาต้นทาง
  └── ยกเลิก (ไม่เสียหาย) → OOP > ปิดงาน > กลับคืนสาขา
```
- ปิดงานได้เฉพาะ: เกิน SLA + ถูกเคลม/ชดเชยแล้ว หรือเซ็นรับพัสดุแล้ว
- ยกเลิกตีกลับ → กดโมฆะคำร้อง
- รหัสปิดงานเข้าได้เฉพาะคนที่เกี่ยวข้อง

#### Re-print Label
- Re-print ได้สูงสุด **10–15 ครั้ง**
- บลูทูธ printer → ถ้ามีปัญหาต้องดู CCTV เท่านั้นถึงจะรู้ว่าใครรีปริ้น
- ต้องการ Recheck → กด Re-print ในระบบ

#### กรณีพิเศษที่ต้องจัดการใน JMS
| กรณี | วิธีจัดการ |
|---|---|
| ตีกลับแต่ไม่ปริ้นใบตีกลับมา | ไม่มีใครรู้จนถึงขาออก (SLA ตีกลับ **8 วัน = 5+3**) |
| ใบนำส่งเสียหาย มีเลข | Re-print ส่งต่อได้เลย |
| ใบนำส่งเสียหาย ไม่มีเลข | CC ตามหาเลข → ปิดงาน |
| กระสอบของขาด | ตามหาเลข → ปิดงาน |
| รถ GW | ตีกลับไม่ได้ (รถต้องรับงานกลับด้วย) → ดึงเลขจากใบมอบหมายงาน |
| สาขาเสียหาย >30% | ตีกลับทั้งคัน สาขา repack ใหม่เอง |
| Cancel order | ~**800 ชิ้น/วัน** ปัจจุบัน — สแกนส่งไม่ได้ |

### 4. **บ้านซอง วง D — รายละเอียดเชิงลึก**

#### โครงสร้างโซน
- 4 วง: A, B, C, D
- วง D แบ่งเป็น D1 (หน้า), D2 (กลาง), D3 (หลัง) — แต่ละลานมี **บน/ล่าง**

#### เส้นทางสายพาน
| Line | วิ่งผ่าน | รับงาน |
|---|---|---|
| **Line 5** | D1ล่าง → D3ล่าง → D3บน → D2ล่าง → D2บน | D1ล่าง, D3ล่าง, D3บน |
| **Line 8** | D1บน → D2 | D1บน, D2 |

- กระจายงาน: **D1 ก่อน** → D2 → D3 ตามปริมาณ
- รอบเย็น เริ่ม 17:00 → ต้องเสร็จ **23:37**

#### Auto Packing
- KPI: **1,600 ชิ้น/ชั่วโมง**
- สลับคนทุก 1 ชั่วโมง, สลับตำแหน่งสัปดาห์ละครั้ง (ดู performance)
- หน้าจอแสดง: รูปพัสดุ, เลข, เวลาสแกน, จำนวน, performance

**ปล่อยได้:** กล่อง, ซอง, ถุงพัสดุ, พื้นผิวเรียบ, ≤3 kg และ ≥0.02 (เบาเกิน = ปลิวเข้าช่องผิด)

**ปล่อยไม่ได้:** ทรงกลม, ทรงกระบอก, ซองเล็กบาง, ของขนาดใหญ่/ยาวเกิน

**ขั้นตอนปล่อยงาน:**
1. หงายพัสดุ วางบนเครื่องชั่ง (เสียง Ping)
2. เครื่องอ่านน้ำหนัก + บาร์โค้ด
3. วางลงถ้วยดำบนขบวนรถไฟ
4. ผ่านอุโมงค์ไฟ (อ่านว่าต้องไปบ้านไหน)
5. วนสูงสุด **3 รอบ** → ตกช่องที่กำหนด

### 5. **รอบรถ / ทีม Admin (ข้อมูลจาก 12/06)**

**ขั้นตอนการปล่อยรถ:**
1. เช็ครอบออก + คำนวณว่าของขึ้นไปทัน → ปล่อยรถ
2. โทรตามคนขับ (ยืนยัน: ทะเบียนตรง, จังหวัดตรง, เส้นทางตรง)
3. บิลเบิกงาน (ให้คนโหลด) + ใบนำส่ง (ให้คนขับ) → **ต้องเช็คทะเบียนตรงกัน**
4. โหลด 80% → เรียกรถคันต่อไป
5. เช็คเลขสายรัดอันสุดท้าย (internal control)
6. ถ่ายรูปตอนปล่อยรถ (ป้องกันทุจริต)

**การคำนวณตัดรอบ:**
- ดูปริมาณงาน + เวลาถึง + % งานที่ได้ ≥ 80%
- งานเยอะ → อาจปล่อยเลทเพื่อให้ได้ % งานสูงสุด
- ตู้แม่เต็มก่อนลูกเสมอ / ของหนักไปช่องลูก

**Admin = ทีมตามรถ / ปล่อยรถ / โบกรถ / เช็คเอกสาร**
