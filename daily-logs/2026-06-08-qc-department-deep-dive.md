### 2. โครงสร้างแผนก QC SPK

QC (Quality Control / 质控部门) มีจุดประสงค์หลัก 3 ข้อ:
- ตรวจสอบการทำงานให้เป็นไปตามเป้าหมาย
- ตรวจสอบมาตรฐานและปัญหาที่เกิดขึ้น
- ฟีดแบคปัญหาไปยังฝ่ายปฏิบัติการเพื่อแก้ไข

แบ่งออกเป็น 5 หน่วยย่อย:

| หน่วย | หน้าที่ |
|-------|---------|
| Call Center | รับสาย, ตรวจสอบพัสดุลูกค้า, อนุญาโตตุลาการ/เคลม, ตรวจสอบพัสดุตกค้าง SLA 36/48H |
| CCTV | ตรวจสอบการทำงาน Real-time, ตรวจสอบหลักฐานและติดตามพัสดุ |
| Data | ตรวจสอบการทำงานสาขา, ค่าปรับ |
| Packing | จัดการพัสดุที่มีปัญหา |
| Store | เก็บพัสดุที่ไม่มี AWB |

### 3. Flow สถานะพัสดุในระบบ JMS

พัสดุทุกชิ้นต้องผ่านสถานะตามลำดับ ซึ่ง QC ใช้ติดตามและตรวจสอบ:

```
สาขาต้นทาง
  → รับพัสดุ → สแกนบรรจุกระสอบ → สแกนนำส่ง
GW ต้นทาง
  → สแกนถึงลงรถ → สแกนบรรจุกระสอบ → สแกนนำส่งขึ้นรถ
GW ปลายทาง
  → สแกนถึงลงรถ → สแกนบรรจุกระสอบ → สแกนนำส่งขึ้นรถ
สาขาปลายทาง
  → สแกนถึงลงรถ → สแกนเปิดกระสอบ → สแกนนำจ่าย → เซ็นรับ
```

### 4. KPI ทั้ง 5 ตัวของ QC

| # | KPI | คำนิยาม | เป้าหมาย |
|---|-----|---------|---------|
| 1 | ขาเข้า/ขาออกทันเวลา (进出港及时率) | พัสดุสแกนถึงและนำส่งออกภายในรอบรถที่กำหนด (เวลาถึง +60 นาที) | ≥ 99.2% |
| 2 | คัดแยกผิด (错分拣) | พัสดุถูกคัดแยกไปผิดปลายทาง เกิดจากคัดแยกนำส่งผิด / ติดลาเบลกระสอบผิด / ทำงานไม่ได้มาตรฐาน | < 800 ชิ้น/วัน (0.28%) |
| 3 | หลุดสแกน (落扫) | พัสดุไม่ถูกสแกนครบทุกสถานะ ทำให้ tracking ขาดหาย | < 0.4% |
| 4 | เคลียคลัง (清场) | พัสดุที่เข้า 13:00–04:00 ต้องออกก่อน 06:00 / 08:00 / 11:00 (ดึงข้อมูลที่ 6:40 / 8:40 / 11:40) | < 1% |
| 5 | Transit ล่าช้า (中转超时) | พัสดุที่เซ็นรับล่าช้าเกิน SLA | รอยืนยันเป้าหมาย ⚠️ |

## Key Insights

✅ **QC เป็นหน่วยกลางที่คุม 3 มิติพร้อมกัน** — Real-time (CCTV), Data (ย้อนหลัง), และ Customer-facing (Call Center) ทำให้มองเห็นปัญหาได้รอบด้าน

✅ **รหัสคัดแยก HUB vs GW ต่างกัน** — GW ใช้รหัส 999xxx, HUB ใช้รหัสขึ้นต้นด้วยตัวเลขอื่น (9xxxxx) และมีรหัสย่อ 2 ตัวอักษร

✅ **เคลียคลังดึงข้อมูลช้ากว่า deadline 40 นาที** — น่าจะเผื่อเวลา system update แต่ยังไม่ได้ยืนยัน
