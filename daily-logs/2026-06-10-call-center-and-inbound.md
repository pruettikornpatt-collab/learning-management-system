## What I Learned

### 1. Call Center 1470 — ภาพรวมและขอบเขตงาน

**หน้าที่หลัก:** รับสายและติดตามพัสดุให้ลูกค้าที่โทร 1470 เท่านั้น — ถ้าไม่ได้โทรผ่าน 1470 ไม่ใช่หน้าที่ CC

**Performance Target:**
- สายหลุด ต้องไม่เกิน 3% แต่ปัจจุบันทำได้จริงอยู่ที่ 50–60%

**Routing เบอร์โทรกลับ:**
ระบบจะส่งเบอร์ให้ลูกค้าตาม **ที่อยู่ล่าสุดของพัสดุในระบบ** ซึ่งอาจล่าช้ากว่าความเป็นจริง เช่น ของถึงขอนแก่นแล้วแต่ยังไม่ได้สแกนรับ → ข้อมูลในระบบยังอยู่ SPK → เบอร์ติดที่ SPK

**SLA ติดตาม:**
- พัสดุ BKK ต้องติดตามให้จบภายใน 48 ชั่วโมง
- กรณีพัสดุหาย: ติดต่อเขตพร้อมหลักฐาน CCTV

**การติดต่อ:** Gateway สามารถติดต่อสาขาได้โดยตรง — กลไกคือ เมื่อลูกค้าติดตามพัสดุแล้วสถานะค้างอยู่ที่ Gateway → Gateway หาหลักฐานจาก CCTV ยืนยันว่าของออกจากคลังไปแล้ว (protect ตัวเอง) → เมื่อพิสูจน์แล้ว ความรับผิดชอบจึงตกอยู่ที่สาขา → Gateway ติดต่อสาขาโดยตรงเพื่อติดตามต่อ — ส่วน 1470/HQ มีอำนาจสูงสุด ติดต่อได้ทุกระดับ
> *(แก้ไข 12 มิ.ย.: เดิมจดว่า Gateway ต้องผ่านเขตเสมอ ติดต่อสาขาตรงไม่ได้ ซึ่งไม่ถูกต้อง)*

- ใบงานเข้ามาจาก TT (TikTok) ผ่านทางแชท
- ขั้นตอน: ลูกค้า → TT → HQ → SPK cc Platform
- CC ตรวจสอบเลขพัสดุ หาสาเหตุ แล้วแจ้งลูกค้ากลับ
- ส่วนใหญ่เป็นพัสดุตกค้างเกิน 48 ชม. หรืออยู่ระหว่างเดินทาง 36 ชม.
  - ถ้าอยู่ระหว่างเดินทาง → แจ้งลูกค้าว่า "กำลังนำส่ง คาดได้รับตาม SLA"
  - ถ้าออกจาก SPK แล้ว → recheck กระบวนการ + ติดต่อสาขาปลายทาง

### 3. ใบสั่งงานทั่วไป

- CC รับสาย 1470 → จบสายแล้ว → สร้างใบสั่งงานให้แผนกติดตามต่อ
- ใบสั่งงานใช้เพื่อ: สั่งให้แต่ละจุดตรวจสอบสถานะพัสดุของตัวเอง และเป็นหลักฐานยืนยัน (เช่น "ส่งออกไปแล้วนะ")
- หน้าที่: ติดตามพัสดุตกค้าง เร่งนำจ่าย (ต้องตกค้างเกิน 36 ชม. ก่อนถึงจะเร่งได้)

### 4. On Ground: กระบวนการ Inbound แบบละเอียด

**7 ขั้นตอนหลัก:**

| ขั้น | กิจกรรม |
|------|---------|
| 1 | รถเข้ามาถึงที่คลัง |
| 2 | พนักงานสแกนเปิดตู้รถ |
| 3 | นำสายพานขึ้นรถ |
| 4 | ส่งใบนำส่งให้พนักงานประจำเครื่อง DWS |
| 5 | พนักงาน DWS ระบุเวลาที่สินค้าชิ้นแรกเข้า |
| 6 | พัสดุทุกชิ้นต้องผ่านเครื่อง DWS |
| 7 | DWS คัดแยกลงตามเขตต่างๆ อัตโนมัติ |

**Conditions — สินค้าที่ DWS แยกไม่ได้:**

```
DWS ไม่สามารถแยกได้
→ วนบนสายพานด้วย function Reverse (พนักงานปัดด้วยตนเอง → ลงเขต)
→ ถ้ายังระบุไม่ได้ → ท้ายราง (จัดการต่อ)
```

> *แก้ไข 12 มิ.ย.: เดิมจดว่า "สายพาน Voyager" ซึ่งจดมาผิด — มี conveyor เดียว "Reverse" คือชื่อ function ที่ J&T ใช้*

- Manual control เครื่องได้ แต่ต้องได้รับคำสั่งจากหัวหน้าก่อน

## Key Insights

✅ **ข้อมูลในระบบกับสภาพจริงมี Lag**
เบอร์ที่ลูกค้าได้รับขึ้นอยู่กับข้อมูลในระบบ ไม่ใช่ที่อยู่จริงของพัสดุ — ถ้า scan ช้า ลูกค้าโทรผิดเขต

✅ **CC มีขอบเขตงานชัดเจน**
Gateway รับผิดชอบเฉพาะ 3 ประเภทงานเท่านั้น (การติดต่อประสานงานทำได้โดยตรงรวมถึงระดับสาขา)

✅ **DWS เป็น bottleneck ของ Inbound**
พัสดุทุกชิ้นต้องผ่าน DWS — สิ่งที่ระบบแยกไม่ได้คือจุดที่ต้องใช้คนเข้ามาช่วย (function Reverse)
