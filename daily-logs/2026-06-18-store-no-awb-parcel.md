### 2. **⭐️ การค้นหา AWB ก่อนลงทะเบียน (สำคัญมาก — ทำก่อนเสมอ)**

**หลักการ:** ถ้าหาเลขได้ → ส่งต่อ/ตีกลับได้เลย ไม่ต้องลงทะเบียนพัสดุไม่มีAWBเลย

#### วิธีที่ 1 — บันทึกข้อมูลรอบรถ (เมื่อขนถ่ายลงรถแล้วพบ)
- จด: ใบมอบหมายงาน / ทะเบียนรถ / สถานีก่อนหน้า / เลขAWBบางส่วน

#### วิธีที่ 2 — เมนู JMS: ค้นหา AWB ที่เสียหาย (残单速查)
```
เส้นทาง JMS: ประสิทธิภาพการบริการ → การจัดการที่ไม่ปกติ
            → การจัดการพัสดุไม่มีเลขAWB → ค้นหาAWBที่เสียหาย
```
- กรอกข้อมูลแบ่งด้วย Enter: รหัสคัดแยกส่วนที่ 1 → บ้านเลขที่ → ชื่อผู้รับ → เลขAWBที่หลงเหลือ
- ตัวอย่าง: `A1 (Enter) 5/88 (Enter) สายชล (Enter) 745859`
- ใช้ `*` แทนส่วนที่ไม่ชัด เช่น `74585*` หรือ `74*859`
- รหัสคัดแยกแบ่ง 4 ส่วน (ส่วนที่ 1 = คลัง/เขต, 2 = เส้นทาง/รอบรถ, 3 = สาขาปลายทาง, 4 = ลำดับย่อย)
- 2 วิธีกรอก: **填空录入** (กรอกตรงตามช่อง) หรือ **智能检索** (พิมพ์ไม่ต้องเรียงลำดับ แบ่งด้วย Enter)

#### วิธีที่ 3 — เลขตกค้าง
- ดึงเลขที่สแกนถึงแล้วยังไม่ส่งออก → วางในช่อง "เลขAWBที่เข้าข่าย" ได้สูงสุด **500 เลขต่อครั้ง**
- ระบบจับคู่ให้อัตโนมัติ → ตรวจสอบและส่งออก

#### วิธีที่ 4 — เลขสั่งซื้อต่างประเทศ ขึ้นต้น TH
- ค้นหาผ่านเมนู "ติดตามพัสดุ" → "อิงตามหมายเลขคำสั่งซื้อ" → **ไม่ต้องลงทะเบียน**

#### วิธีที่ 5 — Order ID พัสดุ TikTok
- ใส่ Order ID ในเมนูค้นหา AWB ที่เสียหายได้โดยตรง

#### วิธีที่ 6 — สแกน QR code ด้วยมือถือ
- ใช้เมื่อ PDA ไม่สแกน barcode ได้

#### วิธีที่ 7 — AI จับคู่รูปภาพสินค้า (无头件智能检索功能)
- อัปโหลดรูปสินค้าได้สูงสุด **200 รูป** ต่อครั้ง
- ระบบ AI จับคู่จากภาพ → เลือกกรอง: สาขาที่ลงทะเบียน, เวลา, สถานะ

#### วิธีที่ 8 — จับคู่ตัวอักษร
- บรรยายลักษณะสินค้า เช่น "ขวด สีดำ กระสอบ ถัง แกลลอน"

### 3. **การลงทะเบียนพัสดุไม่มีAWB ใน JMS / APP**

**ใช้เมื่อ:** ค้นหาด้วยทุกวิธีแล้วยังไม่พบเลข AWB

**เส้นทาง JMS:**
```
ประสิทธิภาพการบริการ → การจัดการที่ไม่ปกติ
→ การจัดการพัสดุไม่มีเลขAWB → เลือก "ลงทะเบียนพัสดุไม่มีเลขAWB"
```

**ข้อมูลที่ต้องกรอก 11 รายการ:**

| # | ข้อมูล | หมายเหตุ |
|---|---|---|
| 1 | น้ำหนัก (กก.) | ทศนิยม ≤2 ตำแหน่ง, ชั่งจากบรรจุเดิมเท่านั้น |
| 2 | ชนิดสินค้า | ดูตารางประเภทสินค้า |
| 3 | ยาว × กว้าง × สูง | หน่วย CM |
| 4 | ชื่อสินค้า | ทราบยี่ห้อ → `(โอโม่)ผงซักฟอก 2000g` / ไม่ทราบ → `(N/A)เสื้อผ้าลายการ์ตูน` — **ห้ามใช้คำว่า "ไม่ทราบ" หรือ "ไม่ชัดเจน"** |
| 5 | สินค้าสมบูรณ์? | ใช่ = บรรจุภัณฑ์ OK / ไม่ใช่ = ไม่แน่ใจจำนวนสินค้า |
| 6 | สาเหตุไม่มีAWB | ใบชำรุด / ข้อมูลไม่ชัด / ไม่มีใบ / หลุดออก / บริษัทอื่น / อื่นๆ |
| 7 | จำนวนสินค้า | นับครบทุกชิ้น |
| 8 | สถานีก่อนหน้า | สาขา/SC/GW/HUB ที่พัสดุมาจาก |
| 9 | ป้ายทะเบียนรถ | บันทึกทะเบียนที่ขนมาส่ง + ใบมอบหมายงาน |
| 10 | ขั้นตอนที่พบ | ภายในกล่อง / กำลังคัดแยก / ขนถ่ายลงรถ / ขนถ่ายขึ้นรถ |
| 11 | แหล่งที่มา Platform | Shopee / Lazada / JMS / TikTok ฯลฯ |

**การถ่ายรูป (บังคับอย่างน้อย 3 รูป จากบรรจุเดิมเท่านั้น):**

| รูปที่ | เนื้อหา |
|---|---|
| 1 | ภาพพัสดุโดยรวม |
| 2 | ภาพสินค้าภายใน ครบทุกชิ้น |
| 3 | ภาพขณะชั่งน้ำหนัก (เห็นตัวเลขชัดเจน) — **ชั่งจากบรรจุเดิมเท่านั้น ห้ามหีบห่อใหม่ก่อนชั่ง** |
| 4-5 | รายละเอียดใบนำส่ง หรือข้อมูลอื่น (ถ้ามี) |

**วิดีโอ:** ไฟล์ไม่เกิน 5MB / สูงสุด 3 วิดีโอ ≤50MB รวม / ความยาว ≤10 นาที

### 4. **สถานะพัสดุไม่มีAWB และกระบวนการยืนยัน**

```
ลงทะเบียน + กดเผยแพร่
        ↓
[รอการเรียกร้อง] ← สาขาต้นทางยังไม่ยืนยัน
        ↓  สาขาต้นทางกดยืนยัน
[กำลังเรียกร้อง] ← ฝ่ายลงทะเบียนต้องตรวจสอบภายใน 24 ชม.
        ↓  ฝ่ายลงทะเบียนกดยืนยัน
[เสร็จสิ้นแล้ว] → ปริ้น AWB → ส่งออก/ตีกลับ
```

**สถานะอื่น:**
- `ยกเลิกแล้ว` = ลงซ้ำ หรือส่งออกแล้ว
- `เป็นโมฆะ` = ครบกำหนดเก็บแล้วไม่มียืนยัน

**ฝ่ายลงทะเบียนส่งเลขเข้าข่ายไปสาขาต้นทาง:**
1. กดแก้ไข → กรอกเลขAWBที่เข้าข่าย → ระบบดึงสาขาต้นทางอัตโนมัติ
2. กรอกสถานีก่อนหน้า → กดเผยแพร่
3. สาขาต้นทาง**ต้องตรวจสอบทุกวัน** ที่ช่อง "บันทึกแจ้งเตือนที่เข้าข่าย"

**เลขปิดเคลมแล้ว:**
- สาขาต้นทางเลือก "ทำลายทิ้ง" → ฝ่ายลงทะเบียนกด "เปลี่ยนเป็นพัสดุโมฆะ" + แนบภาพหลักฐาน

### 5. **การแพ็คส่งออกตามประเภทสินค้า**

| ประเภทสินค้า | วิธีแพ็ค |
|---|---|
| ของเหลวชนิดขวด | บับเบิ้ล 3 ชั้นทุก 6 ด้าน, กล่องหนา ≥5mm, เทปรูป H 2 ชั้น, **วางปากขวดขึ้น** |
| ของเหลวชนิดถุง | บับเบิ้ล 3 ชั้นคลุมทั้งหมด, กล่องหนา ≥5mm, เทปรูป H 2 ชั้น |
| กล่องทั่วไป | บับเบิ้ล 3 ชั้น, เทปรูป H 2 ชั้น |
| กล่องเครื่องดื่ม | บับเบิ้ล ≥3 ชั้นภายนอกกล่องด้วย, กล่องลูกฟูก ≥5 ชั้น |
| กระสอบ | กรณี1: คลุมกระสอบแข็งหนาเพิ่ม / กรณี2: ใส่กล่อง + บับเบิ้ล 2 ชั้น + เทป + |
| ของแตกหักง่าย | บับเบิ้ล ≥3 ชั้น, กล่องเหมาะสม, บับเบิ้ลกันกระแทกเพิ่มจนไม่มีที่ว่าง |
| ผลไม้ | บับเบิ้ล 3 ชั้น + **เจาะรูระบายอากาศ** |
| ต้นไม้ | ห่อรากด้วยถุงพลาสติก, พันลำต้น 3 รอบ, กล่องแนวตั้ง + เจาะรู + ลูกศรชี้บน |

### 6. **พัสดุไม่มีAWB เปลี่ยนเป็นโมฆะ — 2 กรณี**

- **กรณีที่ 1:** ครบกำหนดจัดเก็บ สถานะยังเป็น "รอการเรียกร้อง" → กดเปลี่ยนเป็นโมฆะ (แนบรูปพัสดุทั้งชิ้นชัดเจน รวมถึงวันหมดอายุถ้ามี)
- **กรณีที่ 2:** พบเลขพัสดุแล้ว แต่เลขนั้นปิดเคลมสูญหายแล้ว → สาขาต้นทางเลือก "ทำลายทิ้ง" → ฝ่ายลงทะเบียนแนบภาพยืนยัน → กดเปลี่ยนเป็นโมฆะ

### 7. **ใบสั่งงานควบคุม (无头件工单) — บทลงโทษ**

```
เส้นทาง JMS: ประสิทธิภาพการบริการ → ใบสั่งงานการควบคุม
            → การยื่นเรื่องใบสั่งงาน → ยื่นรายงาน
```

| รหัส | กรณี | ปรับ (ครั้ง/เดือน) | สูงสุดต่อวัน |
|---|---|---|---|
| DP09001 | ลงทะเบียนไม่ได้มาตรฐาน (ข้อมูลผิด/ไม่ครบ) | 10 | 5 |
| DP09002 | รายงานไม่ได้มาตรฐาน (ภาพไม่ชัด/ไม่ครบ) | 30 | 5 |
| DP09003 | พัสดุในคลังไม่ได้ลงทะเบียน | 100 | 50 |
| DP09004 | เจตนาไม่ยืนยัน (สาขาต้นทางไม่ยืนยันใน 24 ชม.) | 500 | — |

**4 กรณีต้องรายงาน:**
1. **รายงานไม่ได้มาตรฐาน** — ภาพไม่ชัด, ข้อมูลไม่ครบ
2. **รายงานเท็จ** — มีเลขAWBอ่านได้ชัดแต่ยังลงทะเบียนเป็นไม่มีAWB
3. **พัสดุในคลังไม่ได้ลงทะเบียน** — พบพัสดุไม่มีAWBในคลังแต่ไม่ได้ลงในระบบ
4. **เจตนาไม่ยืนยัน** — สาขาต้นทางไม่ยืนยันหลังแจ้ง 24 ชม. (ตั้งแต่ มี.ค. 68 HQ เป็นผู้ลงร้องเรียนเอง)

### 8. **มุมมองของ Store department (จาก Presentation)**

Store รับพัสดุไม่มีAWB จาก SC (Sorting Center) ทำงาน 3 ขั้นตอนหลัก:

```
1. ลงทะเบียน JMS
   ↓ เช็คจำนวน + เก็บข้อมูลแต่ละจุดที่ส่งมา
   ↓ ถ่ายรูป 3-5 รูป + วัดขนาด + ชั่งน้ำหนัก
   ↓ เก็บแยกตามประเภท + เรียงตามวันที่

2. แมทพัสดุ
   ↓ ดึงเลขตกค้างออกมาทั้งหมด
   ↓ เช็คชื่อ/สี/ขนาด/น้ำหนัก/ยี่ห้อ/เบอร์/ไซส์ให้ตรงกับพัสดุจริง
   ↓ ลงทะเบียนพัสดุมีปัญหา → รีแพ็ค → ส่งออก

3. ยืนยันพัสดุที่มาในระบบ
   ↓ ค้นหาจากเลขลงทะเบียนพัสดุไม่มีAWB
   ↓ ลงทะเบียนพัสดุมีปัญหา → รีแพ็ค → ส่งออก
   ↓ ส่งเข้าฝ่าย OP คัดแยกต่อ
```

## Key Insights

✅ **ค้นหาก่อนลงทะเบียนเสมอ — ประหยัดเวลามาก**
มี 8 วิธีค้นหา AWB ก่อนที่จะต้องลงทะเบียน ถ้าค้นเจอ → ส่งต่อได้เลย ไม่ต้องทำทั้ง workflow ลงทะเบียน-ยืนยัน-ส่งออก ซึ่งกินเวลาและต้องรอสาขาต้นทางด้วย

✅ **ชื่อสินค้าเป็น audit trail สำคัญ**
ห้ามใช้ "ไม่ทราบ" หรือ "ไม่ชัดเจน" เพราะข้อมูลนี้ใช้ AI จับคู่รูปภาพ และใช้เป็นหลักฐานยืนยันกับสาขาต้นทาง ถ้ากรอกผิดหรือกว้างเกิน → จับคู่ไม่ได้ → พัสดุค้างนานขึ้น

✅ **บทลงโทษ DP09004 รุนแรงที่สุด**
ปรับ 500 ครั้ง/เดือน สำหรับสาขาต้นทางที่เจตนาไม่ยืนยัน และตั้งแต่ มี.ค. 68 HQ ดึงข้อมูลและลงร้องเรียนเองโดยอัตโนมัติ — แสดงว่าปัญหานี้เคยเกิดบ่อยมากพอที่จะต้องทำ automation

✅ **น้ำหนักต้องชั่งจากบรรจุเดิมเสมอ**
ถ้าชั่งจากบรรจุใหม่ = น้ำหนักไม่ตรงกับที่ลงระบบ → ถือว่าการลงทะเบียนไม่ได้มาตรฐาน (DP09001)

## Connection to Management

กระบวนการพัสดุไม่มีAWB สะท้อน **ความสำคัญของ data quality ที่ต้นทาง** — ถ้าสาขาต้นทางแปะใบนำส่งครบและถูกต้องตั้งแต่แรก พัสดุกลุ่มนี้จะไม่เกิดขึ้นเลย ทั้ง workflow นี้จึงเป็น "cost of poor quality" ที่แท้จริง ทุกชิ้นที่ผ่านกระบวนการนี้ใช้ทรัพยากร: คนลงทะเบียน / คนแมท / คนรีแพ็ค / พื้นที่คลัง / เวลารอยืนยัน

บทลงโทษ DP09003 (100 ครั้ง/เดือน) และ DP09004 (500 ครั้ง/เดือน) แสดงให้เห็นว่า J&T ใช้ **negative incentive** ในการบังคับ compliance แทนที่จะพึ่ง goodwill — เป็น design ที่ realistic สำหรับ network ขนาดใหญ่ที่มีสาขาจำนวนมากและ alignment ยาก
