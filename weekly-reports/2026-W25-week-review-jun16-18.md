## Big Learnings ของสัปดาห์นี้

### 1. ลำดับชั้นของคลังและกฎความรับผิดชอบ (16/06)

J&T มี 3 ระดับคลัง — GW (Gateway), HUB, และสาขา — โดยมีกฎชัดเจนว่าใครส่งให้ใครได้:

| คลัง | ส่งได้ถึง |
|---|---|
| GW | GW, HUB, สาขา |
| HUB | GW, HUB (ต่างจังหวัดส่งสาขาได้ด้วย) |
| สาขา | ลูกค้าปลายทาง |

กฎที่สำคัญที่สุด: **รถ GW → GW ตีกลับไม่ได้เลย** และ **สาขาที่มีพัสดุมีปัญหาเกิน 30% ตีกลับทั้งคันได้ สาขารับผิดชอบ repack เอง** — กฎสองข้อนี้สร้าง accountability และแรงจูงใจในการควบคุมคุณภาพตั้งแต่ต้นทาง

### 3. รหัสคัดแยก (Sorting Code) และระบบ JMS Packing (17/06)

รหัสคัดแยกแบ่ง 4 ส่วน ได้แก่ คลัง/เขต, เส้นทาง/รอบรถ, สาขาปลายทาง, และลำดับย่อย รหัสสำคัญที่ต้องจำ: รหัส `00` = อ่านไม่ได้ ไปท้ายราง / รหัส `*` = คัดแยกผิดหรือสาขาปิดงานแล้ว / หลุดสแกนรับ = สแกนส่งออกไม่ได้

**Flow พัสดุมีปัญหา (Packing ใน JMS):**
```
พัสดุมีปัญหา → "ระงับ"
  ├── เสียหายทั้งหมด → รอ HQ approved → ปิดงาน
  ├── เสียหายบางส่วน → ถามลูกค้า → ซ่อมส่ง / ตีกลับ
  └── ยกเลิก → OOP → ปิดงาน → คืนสาขา
```

Cancel order ปัจจุบัน **~800 ชิ้น/วัน** ซึ่งสแกนส่งออกไม่ได้ทั้งหมด เป็น hidden workload ของทีม Packing

### 6. SOP พัสดุไม่มี AWB (无头件) — 54 หน้า (18/06)

นี่คือ learning ที่หนักที่สุดของสัปดาห์ SOP ครอบคลุมทุกกรณีตั้งแต่พบพัสดุจนถึงส่งออกหรือเปลี่ยนเป็นโมฆะ

**หลักการสำคัญ: ค้นหา AWB ก่อนลงทะเบียนเสมอ** มี 8 วิธีค้นหา ได้แก่ บันทึกข้อมูลรอบรถ / เมนู JMS ค้นหา AWB เสียหาย / เลขตกค้าง / เลขขึ้นต้น TH / Order ID TikTok / QR code มือถือ / **AI จับคู่รูปภาพ** (200 รูป/ครั้ง) / จับคู่ตัวอักษร ถ้าค้นพบเลขได้ → ส่งต่อได้เลย ไม่ต้องลงทะเบียน

**ข้อมูล 11 รายการที่ต้องกรอกเมื่อลงทะเบียน** (น้ำหนัก, ชนิดสินค้า, ขนาด, ชื่อสินค้า, สภาพ, สาเหตุ, จำนวน, สถานีก่อนหน้า, ทะเบียนรถ, ขั้นตอนที่พบ, Platform)

**ข้อห้ามสำคัญ:** ห้ามชั่งน้ำหนักจากบรรจุภัณฑ์ใหม่ ต้องชั่งจากบรรจุเดิมเท่านั้น / ห้ามใช้คำว่า "ไม่ทราบ" หรือ "ไม่ชัดเจน" ในชื่อสินค้า

**บทลงโทษ:**

## Patterns ที่สังเกตเห็น

**1. Escalation Structure ที่ชัดเจน:** ปัญหาเล็กแก้ได้เลยที่หน้างาน ปัญหากลางต้องให้สาขาต้นทาง confirm ปัญหาใหญ่ต้องรอ HQ อนุมัติ VIP ต้องให้สาขาปลายทางตัดสิน — แต่ละระดับมีความรับผิดชอบชัดเจนไม่ซ้อนกัน

**2. Negative Incentive เป็น Default Compliance Mechanism:** ค่าปรับ DP09003-DP09004 ที่ automation เพิ่มเข้ามา แสดงว่า J&T ไม่พึ่ง goodwill แต่ใช้ negative incentive บังคับ compliance ซึ่ง realistic สำหรับ network ขนาดใหญ่

**3. Data Quality at Source = Cost of Poor Quality:** ทั้ง SOP พัสดุไม่มี AWB คือ cost ที่เกิดจากต้นทางไม่ติดใบนำส่งให้ถูกต้อง ทุกชิ้นที่ผ่านกระบวนการนี้กินทรัพยากรหลายอย่าง ถ้า upstream quality ดีพอ workflow ทั้งหมดนี้ไม่ต้องมีเลย

**4. AI เริ่มเข้ามาใน Logistics จริงๆ:** Feature AI จับคู่รูปภาพ 200 รูป/ครั้ง ใน JMS บอกว่าปัญหาพัสดุไม่มี AWB มีปริมาณสูงพอที่ต้องสร้าง feature นี้ขึ้นมา ไม่ใช่แค่ manual
